8. Component Overhaul: "Readiness Check" & Combined Mock Exam System

System Architecture Changes

* Page Consolidation: The standalone Exam Reviewer page is completely removed. Its core diagnostic and comprehensive questioning features are integrated directly into this updated module.

* Renaming: The component and page headers must now display "Readiness Check" instead of "Readiness".

* Visual Refactoring: Remove the constraining white background container where the section used to sit. The "Interactive Readiness Check" section must now stretch directly onto the parent layout background, adopting the wide, elegant footprint of that original container.

UX & Feature Configurations
Option 1: Gamified Quick Diagnostics (Dynamic Setup)
* Subject Options (5 Choices): General Education, Professional Education, Mathematics, Science, and the newly added "Logical Reasoning" option.

* Item Count Restructuring: The restrictive 5-item mode is completely deprecated. Selection scales are exactly: 10 items, 15 items, 20 items, and 25 items. This mode retains full interactive animations and gamified micro-interactions.

* Difficulty Scaling: Retains its dynamic sliders/toggles (Easy, Medium, Hard).

Option 2: Full-Scale Comprehensive Mock Exam (Realistic Simulation)
* Access Point: Accessible under the difficulty/level selection panel via a clear call-to-action button to initiate a full simulation.

* Fixed Scope: A massive 250-item exam block composed of exactly 50 items per subject across all 5 subjects.

* Dedicated Layout: Redirects the user to a dedicated simulation wrapper interface.

* Core Systems: Features a live structural countdown timer widget, a persistent subject-navigation sidebar tracking question completion per subject layout (instead of completely randomized question structures), and state tracking.

Global Post-Exam Feedback System
* Upon completing either Option 1 or Option 2, the UI must replace the active viewport with a personalized diagnostic card reporting their final scores, breakdown per subject, and a tailored readiness summary validating how prepared they are to challenge real-world scholarship examinations.



### 📱 Responsive Layout Checklist

* **Grid Breaks:** Ensure the main configuration split uses `grid-cols-1 lg:grid-cols-12` so the panels stack vertically on mobile screens and snap into a clean side-by-side alignment on wider screens.
* **Touch Targets:** The subject buttons and answer option elements include ample padding (`p-3.5` to `p-4`) and comfortable row gaps (`gap-3`) to prevent accidental misclicks on smaller touch screens.
* **Matrix Accessibility:** The 50-item question navigation block utilizes a scrolling container (`max-h-40 overflow-y-auto`) to keep the sidebar from stretching infinitely on mobile viewports.
* **Ambient Uniformity:** By removing the explicit white bounding background, the component will seamlessly adapt to the overarching background scaling of your app container, eliminating awkward margins on narrow screen breaks.
