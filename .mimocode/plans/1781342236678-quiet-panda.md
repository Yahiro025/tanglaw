### Issue 4: Long Main-Thread Tasks (4 tasks found)
**Audit**: "Avoid long main-thread tasks"

Long tasks (>50ms) block the main thread and increase TBT.

#### Step 4.1: Defer NatureCanvas initialization
**File**: `frontend/src/components/nature-canvas.tsx`
- Currently runs immediately on page load
- Add `requestIdleCallback` or `setTimeout(() => {...}, 0)` to defer the heavy initialization
- The canvas setup (particle creation, glow cache, background cache) is likely one of the long tasks

#### Step 4.2: Defer EtheralShadow SVG filter setup
**File**: `frontend/components/ui/etheral-shadow.tsx`
- The SVG feTurbulence filter creation and animation setup could be deferred
- Use `requestIdleCallback` for the IntersectionObserver setup

#### Step 4.3: Break up long tasks with `scheduler.yield()`
**Files**: Any component with heavy initialization
- Use `scheduler.yield()` (available in modern browsers) to break long tasks into smaller chunks
- Or use `setTimeout(fn, 0)` as a fallback

### Issue 5: Non-Composited Animations (1 found)
**Audit**: "Avoid non-composited animations"

Non-composited animations force the browser to repaint entire frames instead of just compositing layers.

#### Step 5.1: Identify the non-composited animation
Run Lighthouse with `--view` flag or check the Performance panel to identify which animation triggers layout/paint.

Likely candidates:
- `filter: blur()` in EtheralShadow or MascotWithGlow
- `backdrop-blur` in site-header
- `transform` + `opacity` animations that also change `width`/`height`

#### Step 5.2: Ensure all animations use compositor-friendly properties
- Only animate `transform` and `opacity` (these are compositor-only)
- Replace `filter: blur()` with `will-change: filter` or a pre-blurred image
- Add `will-change: transform` to animated elements
- Ensure `backdrop-filter` is only used on elements that are already on their own layer

### Issue 6: Image Delivery — Est savings of 43 KiB
**Audit**: "Improve image delivery"

#### Step 6.1: Compress owel-head.png → Lossless WebP
**File**: `frontend/public/assets/owel-head.png` (179KB)
- Convert to **lossless** WebP using: `cwebp -lossless owel-head.png -o owel-head.webp`
- Lossless WebP typically saves 30-50% vs PNG while being pixel-perfect identical
- Update all references in:
  - `frontend/src/components/site-header.tsx`
  - `frontend/src/components/owel-chatbot.tsx`
  - `frontend/src/components/owel-briefing.tsx`
  - `frontend/src/app/layout.tsx` (favicon)

#### Step 6.2: Optimize etheral-mask.webp and etheral-noise.webp
**Files**: `frontend/public/assets/etheral-mask.webp` (205KB), `etheral-noise.webp` (70KB)
- These are already WebP — check if they can be re-encoded at lossless with smaller size
- If original source is available as PNG, convert from PNG to lossless WebP: `cwebp -lossless`
- If already WebP, verify they're at optimal lossless compression: `webp -lossless -m 6` (max compression effort)

#### Step 6.3: Optimize team photos for about page (lossless)
**Files**: `frontend/public/team/*.jpg`, `*.png`
- Several photos are 500KB-1MB (alberto2.0.jpg=1MB, salvaloza2.0.png=1MB)
- Since they're below the fold, ensure `<Image>` uses lazy loading (default) and reasonable sizes
- For PNGs: convert to **lossless** WebP using `cwebp -lossless` (pixel-perfect, ~30-50% smaller)
- For JPGs: convert to **lossless** WebP using `cwebp -lossless` (pixel-perfect, may be similar size or smaller)
- Target: each photo < 80KB if possible, but quality is paramount

#### Step 6.4: Add explicit width/height to prevent CLS on images
**File**: `frontend/src/app/about/page.tsx`
- Verify all `<Image>` components have explicit `width`/`height` or `fill` with aspect-ratio container
