# Plan: Fix "Five Pillars of TANGLAW" to show one pillar at a time

## Problem
The five pillars currently display as a horizontal scroll carousel with 340px-wide cards side by side. The user wants each pillar to display one at a time, taking up the full center screen, regardless of mobile or desktop view.

## File to modify
`frontend/src/app/about/page.tsx` — `CarouselSection` component (lines 192-405)

## Changes

### 1. Replace horizontal scroll carousel with a single-card view
Remove the `overflow-x-auto snap-x snap-mandatory` scroll container. Instead, render only the active pillar card centered on screen.

- Remove `carouselRef` and all scroll-based logic (`getCardWidth`, `handleScroll`, scroll event listener, `isProgrammaticScroll`)
- Keep `activeIndex` state, `nextSlide`, `prevSlide`, `pauseAutoPlay`, autoplay timer
- Render a wrapper `div` that shows only `PILLARS[activeIndex]` with a framer-motion `AnimatePresence` + `motion.div` for enter/exit transitions

### 2. Make the card full-width and centered
Replace the card's `basis-[340px] shrink-0` classes with `w-full max-w-2xl mx-auto` so it takes available space but stays centered and readable.

### 3. Add slide transition animation
Use `AnimatePresence` with `mode="wait"` and `motion.div` with `key={activeIndex}` to animate between slides:
- `initial={{ opacity: 0, x: 40 }}` (enter from right)
- `animate={{ opacity: 1, x: 0 }}`
- `exit={{ opacity: 0, x: -40 }}` (exit to left)

### 4. Keep existing controls
The prev/next arrow buttons and pagination dots stay unchanged — they already work with `activeIndex`.
