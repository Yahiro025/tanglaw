# Speed, Responsiveness & Aesthetic Improvements for TANGLAW

## Context

After comparing with Landas, TANGLAW's particle system is technically superior (physics, interactivity) but the overall UX lacks polish. Landas wins on thematic cohesion, calm pacing, and micro-detail. This plan addresses speed, responsiveness, and aesthetics — with a merged particle system that combines the best of both.

---

## Phase 1: Merge Particle Systems

**Goal:** Combine `nature-canvas.tsx` (physics engine) + `particles-background.tsx` (config layer) into one unified system.

### What each system contributes:
| Feature | From `nature-canvas.tsx` | From `particles-background.tsx` |
|---|---|---|
| Physics engine | ✅ velocity, friction, spring-back | ❌ |
| Mouse interaction | ✅ repulse, wind drift, glow boost | ❌ |
| Click burst particles | ✅ radial burst with color | ❌ |
| Particle trails | ✅ 8-point trail | ❌ |
| Sparkle/twinkle | ✅ cross-pattern sparkle | ❌ |
| Connection lines | ✅ nearby particle links | ❌ |
| Responsive particle count | ✅ hardcoded 75/55/35 | ✅ configurable via props |
| Theme-aware colors | ✅ 7-color palettes, RGB interpolation | ✅ hex color arrays |
| CSS glow filter | ❌ | ✅ drop-shadow via CSS var |
| ParticlesProvider | ❌ | ✅ engine init pattern |
| Dashboard skip | ✅ `usePathname()` | ✅ `usePathname()` |
| Idle/pause detection | ✅ IntersectionObserver + visibility | ✅ idle timer (5s) |
| Reduced motion | ✅ `prefers-reduced-motion` | ❌ |
| LRU glow cache | ✅ 1500 entries | ❌ |
| Offscreen bg cache | ✅ gradient cached | ❌ |
| Frame rate cap | ✅ 30fps | ❌ |

### Merge strategy:
- **Keep `nature-canvas.tsx` as the rendering engine** (it has all the physics, interactivity, caching)
- **Extract configurable props** from tsParticles pattern (particle count by viewport, color scheme, glow filter CSS var)
- **Delete `particles-background.tsx`** and its tsParticles dependencies
- **Update `dynamic-backgrounds.tsx`** to export only the merged system
- **Update `layout.tsx`** to remove the tsParticles component
- **Remove `@tsparticles/*` packages** from `package.json`

### Files to modify:
1. `frontend/src/components/nature-canvas.tsx` — add configurable props, CSS glow filter, idle timer
2. `frontend/src/components/dynamic-backgrounds.tsx` — remove `DynamicParticlesBackground`
3. `frontend/src/app/layout.tsx` — remove `<DynamicParticlesBackground />`
4. `frontend/components/ui/particles-background.tsx` — delete
5. `frontend/package.json` — remove `@tsparticles/engine`, `@tsparticles/react`, `@tsparticles/slim`
6. `frontend/src/__e2e__/verify-particles.spec.ts` — update to test merged system
7. `frontend/tsparticles-fix-spec.md` — archive (spec is now moot)

## Phase 3: Aesthetic — Visual Polish

### 3.1 Slow Down Transitions for Calm Pacing
Landas uses `duration-500` to `duration-700` consistently. TANGLAW uses `duration-200` in many places.

- Buttons: `duration-200` → `duration-300`
- Cards: `duration-300` → `duration-500`
- Nav links: add `duration-500`
- **Files:** `landing-animations.tsx`, `site-header.tsx`, `site-footer.tsx`

### 3.2 Add Decorative Glow Corners to Cards
Landas cards have subtle decorative blur circles in corners. Add to TANGLAW cards:
```tsx
<div className="absolute -right-8 -top-8 h-24 w-24 rounded-full bg-primary/5 blur-3xl" />
<div className="absolute -left-8 -bottom-8 h-20 w-20 rounded-full bg-accent-periwinkle/5 blur-2xl" />
```
- **Files:** `landing-animations.tsx` (FeatureCard), `home-client.tsx`

### 3.3 Add Staggered Entry Animations
Landas hero animates in with staggered delays (subtitle → title → description → buttons → badges).

- Add delay utility classes: `.delay-100` through `.delay-400` with `animation-delay` + initial `opacity: 0`
- Apply to hero elements in `home-client.tsx`
- **Files:** `globals.css`, `home-client.tsx`

### 3.4 Add Scroll-Triggered Section Reveals on About Page
- Wrap major sections in `ScrollReveal` with appropriate delays
- **Files:** `frontend/src/app/about/page.tsx`

### 3.5 Fix GlowingText Hydration Flash
- `GlowingText` renders plain `<span>` on server, then applies glow after mount
- Refactor to use CSS-only glow (always present, no mount-dependent branching)
- **Files:** `frontend/components/ui/glowing-text.tsx`
