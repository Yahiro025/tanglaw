# Fix: "DB error" 500 on Diagnostic Check / Mock Exam

## Root Cause

The error originates at `backend/src/controllers/questionController.ts:98`:

```typescript
} catch (err) {
  console.error(err);
  res.status(500).json({ error: "DB error" });  // hides the real error
}
```

The Prisma query on lines 60-63 or 86-88 is throwing, but the catch block swallows the actual error. The real Prisma exception is only logged to Render server logs.

**Most likely cause**: Schema columns (`assessmentMode`, `isActive`, `sequenceNo`) or the `AssessmentMode` enum don't exist in the actual Supabase database. The schema was updated but `prisma db push` may have failed or the columns were never added.

## Plan

### Step 1: Surface the real error

**File**: `backend/src/controllers/questionController.ts`

Change the catch block (line 96-99) to include the actual error message in the response:

```typescript
} catch (err) {
  console.error("[Questions] error:", err);
  const message = err instanceof Error ? err.message : "Unknown error";
  res.status(500).json({ error: "DB error", detail: message });
}
```

This will let us see the exact Prisma error in the frontend error message and in API responses.

### Step 2: Fix the frontend error display to show detail

**File**: `frontend/src/lib/backend.ts`

In the `authorizedFetch` error handling (around line 90-91), parse the JSON body to extract `detail` if present, so the user (and dev) can see the real error.

### Step 3: Verify and fix schema in Supabase

After deploying Step 1-2, check the actual error. If it's a missing column/enum:

1. Run `cd backend && npx prisma db push` locally against the production DB (using `DIRECT_URL`) to ensure schema is pushed
2. Or verify via Supabase SQL editor that the `Question` table has `assessmentMode`, `isActive`, `sequenceNo` columns and the `AssessmentMode` enum type

### Step 4: Seed questions if table is empty

If the table exists but has no data:
```bash
cd backend && npx ts-node --transpile-only prisma/seed-questions.ts
```

## Critical Files

- `backend/src/controllers/questionController.ts` (error handling + query)
- `frontend/src/lib/backend.ts` (error display)
- `backend/prisma/schema.prisma` (schema source of truth)
- `backend/start.sh` (deploy-time db push + seed)
